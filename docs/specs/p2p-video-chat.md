# P2P ビデオチャット仕様

- 最終確認日: 2026-06-18
- 対象バージョン: 0.1.0

## 目的
PC、タブレット、スマホのブラウザから、URL 共有だけで 1 対 1 の P2P ビデオチャットを開始できるようにする。

## 利用フロー
1. 開始者がトップページを開き、開始ボタンを押す。
2. アプリはカメラとマイクの利用許可を取得し、待機状態になる。
3. アプリは接続用 URL を表示し、開始者はコピーして相手へ送る。
4. 相手は URL を開き、接続ボタンを押す。
5. WebSocket シグナリングで WebRTC の offer / answer / ICE candidate を交換し、P2P 通話を開始する。
6. 終了ボタンまたはページ離脱で通話を終了する。

## 機能要件
- ビデオ表示は自分・相手とも正方形にする。
- マイクミュートとミュート解除を切り替えられる。
- カメラは初回に顔側カメラを優先する。
- 複数カメラがある場合は、ボタン操作で順次切り替える。
- PC、タブレット、スマホの 1 画面内に主要 UI を収め、通常操作でスクロールを不要にする。
- 1 部屋は最大 2 接続に制限する。

## 非機能要件・制約
- GitHub と Cloudflare Workers の連携デプロイを想定する。
- 無料プランで扱いやすい構成として、単一 Worker が HTML と WebSocket シグナリングを提供する。
- Worker のメモリ上に部屋情報を保持するため、 isolate の移動や再起動をまたぐ永続性は保証しない。
- TURN は未設定のため、NAT 環境によっては接続できない場合がある。
